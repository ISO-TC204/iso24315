MDR-10

1 Scope

The management of electronic transport regulations (METR) provides trustworthy, authoritative, machine-interpretable, transport-related rules for using surface transport facilities.

The scope of METR includes both rules that are relatively static (e.g., static speed limits) as well as those that are dynamic (e.g., variable speed limits, lane closures at incident scenes). However, where appropriate, METR is expected to incorporate existing standards (e.g., ISO/TS 19091 for signalized intersections).

This document defines requirements for the data that is to be exchanged among METR component systems. This document is presented in a format that is consistent with the guidance provided by ISO 29148 [2] for a system requirements document. 

The operational concept for METR is provided in ISO 24315-1.

6.5 Discrepancy-related classes
MDR-251

6.5.1 Overview

There are several classes that relate to discrepancies:

  1. A discrepancy fingerprint is used to uniquely identify a specific discrepancy report.
  2. A discrepancy report provides a complete report about a discrepancy as recorded from a field source. It is sent from a METR user to a discrepancy handling system and can be forwarded to a translator upon request.
  3. A consolidated discrepancy report is used by a discrepancy handling system to report a confirmed discrepancy in a summary fashion by omitting or summarizing details and redundancies contained in the individual discrepancy reports.
  4. A rule negotiation request can contain discrepancy reports as a part of its justification.

8.3 Rule location characteristics
MDR-351

8.3.1 Location

Rules are generally associated with locations, which are often referred to as a point location (i.e., a point when the path is viewed in a linear fashion), a linear location (i.e., a stretch along a curvilinear path), or an area location (i.e., an area that encompasses a portion of the surface - or near-surface, such as a bridge or tunnel - of the earth). Area locations can include specific defined areas or an entire jurisdiction. For example, rules might include stop signs or gates (i.e., point locations), on-street parking restrictions (i.e., linear locations), congestion zones (i.e., a custom area location), and rules related to the use of headlights (i.e., jurisdictional area locations).

8.3.2 Heading applicability

Some rules are only applicable to users that have a particular heading. For example, some northbound lanes might only have vehicle occupancy rules in the morning while southbound lanes only have vehicle occupancy rules in the evening.

8.3.3 Map alignment

Some rules need higher location precision to be properly interpreted; mechanisms to provide the required precision need to be conveyed as a part of the rule. For example, while normal GNSS and mapping accuracy of a few meters is sufficient for warnings for animal crossings, parking space information needs typically needs to be accurate to within a few centimeters. Rules that require higher precision need to convey how the location information contained within the rule can accurately be represented into the real-world location (e.g., by identifying a local pavement marking or other landmark that can be used to provide a local reference point).
