DHSR-4

1 Scope

The management of electronic transport regulations (METR) provides trustworthy, authoritative, machine-interpretable, transport-related rules for using surface transport facilities.

The scope of METR includes both rules that are relatively static (e.g., static speed limits) as well as those that are dynamic (e.g., variable speed limits, lane closures at incident scenes). However, where appropriate, METR is expected to incorporate existing standards (e.g., ISO/TS 19091 for signalized intersections).

This document defines the requirements for a METR component system that is responsible for receiving discrepancy reports from METR user systems and providing consolidated, anonymized reports to METR translation systems for further investigation as appropriate. This document is presented in a format that is consistent with the guidance provided by ISO 29148 [2] for a system requirements document. 

For an understanding of how the METR discrepancy handling system fits into the larger METR system of systems, see ISO 24315-3. The operational concept for METR is provided in ISO 24315-2.

6.1.4.1 General

There are three main examples of where this flow is used:

  1. Suppress reports for a METR information item (perhaps with a specific discrepancy type) from any logic system.
  2. Suppress reports for a METR information item (perhaps with a specific discrepancy type) from any release of a logic system prior to a specific release number  (i.e., to filter out reports from an old software version).
  3. To allow reports that were previously cancelled (e.g., after an error in a METR information item was fixed on translator side).

6.2.4.3 Relaying discrepancy

The METR discrepancy handler system shall send a system-generated discrepancy report to each translation system associated with each METR information item for which a discrepancy has been detected.


** The discrepancy report will list the discrepancy fingerprint (i.e., the METR information item id, the error type, the software identifier for the verification detection, and its release date).  **
DHSR-149 NOTE:

The system does not suppress system-generated discrepancy reports. If the system receives an update of METR information containing the same verification error, it will generate a new discrepancy report.
